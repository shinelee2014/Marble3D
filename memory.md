## 📌 1. 本地特有避坑记录

*   **3D 视图中文字在 High-DPI 屏幕上模糊与重影**：
    *   *原因*：Canvas 使用了 `scale(dpr, dpr)` 缩放。若直接使用小数逻辑坐标绘制文本，由于浏览器 subpixel 抗锯齿防抖，会导致文字边缘极度发虚。若阴影偏移量过大（如逻辑 `3px`），会形成虚假“重影”。
    *   *规程要求*：文字坐标必须使用 `Math.round(val * dpr) / dpr` 强制对齐到屏幕物理像素网格。非选中节点的文字阴影偏移量严格设为 `1 / dpr`（即正好 1 物理像素），确保绝对清晰无重影。
*   **3D 文本密集交叉乱码**：
    *   *原因*：当 3D 星云投影到 2D 屏幕时，相邻节点标签极易交叠。
    *   *规程要求*：实现 2D 碰撞检测避让系统（`drawnRegions` 数组记录已画文本范围），发生碰撞时垂直向上推开 `textHeight + 3` 像素，最多尝试 3 次；若依然碰撞，则对非聚焦状态下的普通背景标签执行自动丢弃。
*   **Canvas 频繁修改字体导致的性能瓶颈**：
    *   *原因*：在 `renderLoop` 循环中如果为每个节点动态评估并改变 `ctx.font = ...`，会导致浏览器清空字体渲染缓存，出现卡顿。
    *   *规程要求*：除 Selected/Hover 节点外，普通节点在循环中一律使用常量字符样式 `'10px system-ui, sans-serif'`。
*   **新节点学科分类严格限制**：
    *   *规程要求*：向图谱数据中增加新节点时，必须只能使用已有的学科（数学、语文、英语、科学、信息技术、生活技能、道德与法治）。严禁私自为新节点增设新学科（如把英语节点错写为“词汇与语法”学科）。如确有增加新学科的必要，必须先向用户提议并获得明确同意后方可执行。

## ⚖️ 2. 重大技术决策日志 (ADR)

*   **[2026-07-13] 3D 星云升级为漏斗形知识树**：
    *   *背景痛点*：原 3D 星云支持 360 度任意倾角旋转，导致年级高度坐标倒转或翻转，破坏了“底年级到高年级”的空间垂直递进感。
    *   *最终选型*：重构三维坐标系统，将 **Y 轴设为年级高度轴**（底端一年级 Y=160，顶端六年级 Y=-160），且**分布半径随高度递增而线性放大**（底部紧密抱团，高处发散舒展）。同时将俯仰角 `angleX` 严格锁定在 `[-0.25, 0.6]` 弧度范围内。
    *   *影响约束*：用户拖拽时只能水平环绕自转观察，垂直方向的年级层次（底端为低年级，顶端为高年级）永远保持不变，方向极为清晰。
*   **[2026-07-13] 黄金发光依赖路径绘制**：
    *   *背景痛点*：前置依赖路径不够醒目，缺少重点路径的突显。
    *   *最终选型*：使用 HTML5 Canvas 2D 的 `shadowBlur` 和 `shadowColor` 特效。对于选中节点的前置依赖链，使用高亮金黄色霓虹线加 10px 模糊光圈绘制（Prerequisites）；对于后置解锁线，使用紫色霓虹线绘制。

## 📝 3. 关键业务线与接口备忘

*   **核心 API 接口 (`server.py`)**：
    *   `/api/generate-questions`：AI 仿真与变式题出题接口。接收 `topicId`、`styleHint`（真题风格描述）以及 `extraContext`（原真题题干上下文），支持实时变式出题。
*   **脚本组件 (`scripts/`)**：
    *   `scripts/generate_synthetic_exams.py`：调用本地大模型接口，批量扫描大纲节点，生成 2025/2026 仿真真题并写入 `data/topics_cn.json`。
    *   `scripts/import_and_align_exams.py`：读入本地小升初真题原卷 Word/Text 文档，进行切分、对齐并创建新的大纲挂载节点。
